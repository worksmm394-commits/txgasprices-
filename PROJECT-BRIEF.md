results.

## 8. GitHub Actions Workflows

Both workflows have `workflow_dispatch: {}` → can be manually triggered from the
Actions tab. Both use `concurrency.cancel-in-progress: false` (queue, don't cancel).

### `.github/workflows/update-prices.yml` — hourly
- **Trigger:** cron `0 * * * *` (top of every hour UTC)
- **Timeout:** 10 min
- **Steps:** checkout → Node 20 + npm cache → `npm ci` →
  `node fetch-prices.js` (needs `EIA_API_KEY`, `GASBUDDY_API_KEY`) →
  `node generate.js` → copy `car-db.json` into `output/` →
  `cloudflare/wrangler-action@v3 pages deploy output --project-name=txgasprices`
- **Does not commit** `prices.json` — state-avg refresh is ephemeral and overridden
  each hour.

### `.github/workflows/update-apify-prices.yml` — every 3 days
- **Trigger:** cron `0 10 */3 * *` (10:00 UTC = 5am Central, every 3 days)
- **Timeout:** 15 min
- **Permissions:** `contents: write` (to commit prices.json)
- **Why 3 days:** gas prices change 3–4×/week, so 3-day cadence captures meaningful
  moves while keeping Apify monthly cost ≤ ~$20.
- **Steps:** checkout → Node 20 → `npm ci` →
  `node fetch-apify-prices.js` (needs `APIFY_TOKEN`) → `node fetch-prices.js` →
  `node generate.js` → copy `car-db.json` → Cloudflare deploy →
  commit `prices.json` as `github-actions[bot]` with msg
  `chore: daily Apify price refresh [skip ci]` (the `[skip ci]` prevents trigger
  loop on the hourly workflow).

## 9. `prices.json` Structure

Regenerated by `fetch-apify-prices.js` (full) and `fetch-prices.js` (state avg
refresh). Committed to repo (so the hourly workflow can read the last Apify
snapshot without re-calling Apify every hour).

### Top-level keys
```json
{
  "updated":       "2026-04-16T21:21:37.444Z",   // ISO timestamp of last refresh
  "apifyUpdated":  "2026-04-16T...",              // timestamp of last Apify run
  "stateAverage":  { "regular": 3.685, "midgrade": 4.127, "premium": 4.51, "diesel": 5.247 },
  "source": {
    "aaa":      "gasprices.aaa.com (TX state avg, today)",
    "eia":      null | "EIA API note",
    "gasbuddy": null | "GasBuddy API note",
    "apify":    "apify.com/actors/johnvc~fuelprices (daily, N/M cities, X ZIPs)"
  },
  "note": "Per-chain prices are ESTIMATES derived from the Texas state average plus fixed per-chain offsets...",
  "chains": [ /* fallback chain-level price estimates, see below */ ],
  "cities": { /* per-city Apify live data, see below */ }
}
```

### `chains[]` — statewide chain-level estimates
Used as fallback for cities without Apify coverage.
```json
{
  "chain":     "Murphy USA",
  "priceMode": "estimated",          // "estimated" | "live"
  "regular":   3.565,
  "midgrade":  4.007,
  "premium":   4.39,
  "diesel":    5.147
}
```
Chain offsets defined in `CHAIN_OFFSETS` constant in `fetch-prices.js`.

### `cities[<slug>]` — per-city Apify detail
Keyed by town slug. Present for ~50 cities with Apify coverage.
```json
"houston-tx": {
  "chains": [
    {
      "chain":        "Murphy USA",
      "regular":      3.52,
      "station":      "Murphy USA",
      "address":      "4814 Hwy. 6, Houston",
      "stationCount": 2
    },
    ...
  ],
  "stationsSeen": 30
}
```

`generate.js` prefers `cities[slug].chains` when present, falls back to the
top-level `chains[]` estimator otherwise.

## 10. `towns.json` Structure

- **Location:** repo root
- **Size:** 15 KB, 100-entry array (ordered by population, largest first)
- **Shape:** flat array of objects, no top-level keys
- **Purpose:** master city list. Consumed by `generate.js` to loop over all 100
  cities, by `fetch-apify-prices.js` to decide which ZIPs to scrape, and by
  `buildSitemap()` to emit 5 URLs per city.

```json
[
  { "name": "Houston", "slug": "houston-tx", "county": "Harris",
    "population": 2304580, "lat": 29.7604, "lng": -95.3698 },
  { "name": "San Antonio", "slug": "san-antonio-tx", "county": "Bexar",
    "population": 1434625, "lat": 29.4241, "lng": -98.4936 },
  ...
]
```

### Fields
| Field | Type | Notes |
|---|---|---|
| `name` | string | Display name, e.g. `"Fort Worth"` |
| `slug` | string | URL slug, always suffix `-tx`, e.g. `"fort-worth-tx"` |
| `county` | string | County name without "County" suffix |
| `population` | number | 2020 census approx |
| `lat` | number | Decimal latitude |
| `lng` | number | Decimal longitude |

`slug` is the join key between `towns.json`, `cities-info.json` (object key), and
`prices.json → cities`.
